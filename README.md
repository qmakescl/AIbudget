# 🤖 2026 정부 AI 예산 대시보드

> 41개 부처 · 533개 사업 · 27.4조원 규모의 2026년 정부 AI 예산안을 PDF에서 자동 추출하여 인터랙티브 대시보드로 시각화한 프로젝트

## 📋 프로젝트 개요

정부가 공개한 **2026년도 AI 분야 예산(안) 사업별 설명자료** PDF 문서(41개 부처별)에서 예산 데이터를 자동 추출하고, 이를 분석·시각화하는 대시보드를 구축했습니다.

> 📌 **데이터 출처**: [국가인공지능전략위원회 — AI 예산사업 통합 설명자료](https://www.aikorea.go.kr/web/board/brdDetail.do?menu_cd=000011&num=359&currentPage=1&searchData=&searchText=)

### 주요 기능

- **4개 탭 구조**로 전체사업 / 신규사업 / 예산증가 / 예산감소 사업을 구분하여 분석
- 부처별 예산 규모, 증감 추이, 신규사업 현황 등을 차트와 테이블로 시각화
- 부처별 드롭다운 + 검색 기능으로 원하는 사업 즉시 탐색

## 🛠 개발 과정 (대화 기반)

이 프로젝트는 AI 코딩 어시스턴트와의 **대화를 통해 단계적으로** 만들어졌습니다. 아래는 실제 요청과 그에 따른 구현 과정입니다.

### Step 1: PDF 데이터 추출

- 41개 부처별 PDF(총 5,000+ 페이지)에서 `pdfplumber`와 `PyMuPDF`를 사용하여 예산 테이블 파싱
- 비표준 테이블 구조에 대한 fallback 처리로 533건 100% 추출 달성
- 추출 결과를 `data/budget_data.csv`로 저장

### Step 2: 대시보드 설계 및 구현
>
> *"HTML/JS + Chart.js 로 구성하되 사용자 친화적이고 직관적으로 전체 규모별 부처별로 파악할 수 있게 계획을 수립해줘"*

- KPI 카드 4개 (총 예산, 전체 사업수, 신규사업, 예산 증감)
- 부처별 예산 규모 바 차트 + 예산 변동 도넛 차트
- 부처별 드릴다운 (연도별 추이 + 사업 목록 테이블)

### Step 3: 데이터 정합성 검증
>
> *"pdf 파일에 있는 데이터가 제대로 전달되어 표시하는데 문제가 없는지 검토해줘"*

- CSV → JSON 파이프라인 100% 일치 확인
- PDF 원본과 랜덤 샘플링 대조 (10개 부처 · 20건) → 100% 통과
- 총 예산 단위 변환 오류 수정 (2735.5조 → 27.4조)
- 전년 대비 증감액 계산 로직 수정 (증가사업 합계 → 순증감)

### Step 4: 단위 표기 개선
>
> *"금액 표시할때 단위가 필요하다. 숫자가 많은 편으로 1,000으로 나눠서 단위를 맞춰줘"*
> *"십억단위를 억단위로 처리하자."*

- 모든 금액을 **억원** 단위(백만원 ÷ 100)로 변환
- 차트 Y축, 툴팁, 테이블 헤더에 단위 라벨 추가

### Step 5: 신규사업 상세 분석 추가
>
> *"신규사업들을 위한 상세분석 섹션을 추가해줘"*
> *"부처별 신규사업의 수와 전체 사업 중 신규 사업의 비율도 출력하게 해줘"*

- 부처별 신규사업 예산 바 차트
- 부처별 현황 테이블 (신규 수 / 전체 수 / 비율)
- Top 10 신규사업 차트 + 부처별 필터 목록

### Step 6: 4탭 구조 전면 개편
>
> *"예산 증가 및 감소 사업에 대해도 확인할 수 있게 하고, 이들을 탭으로 구성해줘"*

- **📊 전체사업** / **🆕 신규사업** / **📈 예산 증가** / **📉 예산 감소** 4개 탭
- 각 탭별 부처별 차트 + 현황 테이블 + Top10 차트 + 부처별 필터 목록
- Lazy Rendering으로 성능 최적화

### Step 7: UI/UX 고도화 및 배포 최적화
>
> *"대시보드에 Dark와 Light 테마토글버튼을 추가해줘"*
> *"사업명 위로 마우스오버 하면 사업 내용이 popup 출력되게 해줘"*

- **다크/라이트 🌓 테마 토글**: CSS 변수를 활용한 전역 테마 동적 변경 기능을 구현하고 사용자의 마지막 테마 선택을 `localStorage`에 저장하여 유지
- **사업 내용 툴팁 지원**: 데이터 테이블의 사업명에 마우스를 올리면 (`hover`), 해당 사업의 설명 내용을 담은 말풍선 툴팁 팝업이 우측에 나타나도록 추가. 툴팁 안에서도 텍스트를 드래그하거나 스크롤할 수 있도록 가상 공간(Bridge) 적용
- **커스텀 스크롤바**: 브라우저 기본 스크롤바를 다크/라이트 테마에 맞게 얇고 모던한 느낌으로 통일
- **정적 웹 브라우징 최적화**: API 캐시 타임스탬프를 제거하고 외부 아이콘 폰트(Bootstrap Icons)를 도입하여 GitHub Pages 등 정적 호스팅 서비스(Static Hosting)에서 문제없이 배포 및 동작하도록 개선

### Step 8: 추출 정확도 개선
>
> *"현재 추출에서 문제가 되는 부분을 정리해줘 / 제대로 읽기 위해 코드를 수정하고 데이터를 생성해줘"*

- **사업코드 행 혼입 수정**: `(2047-500)` 형태의 괄호 내 숫자 코드 행이 예산 데이터로 오인식되던 버그 수정
- **6열 테이블 분기 추가**: 경찰청·소방청 등 6열 구조 테이블에서 값이 1칸씩 밀리던 16건 수정
- **6열 3번째 열 자동 판별**: 추경/요구안 중 어느 컬럼인지 헤더 텍스트로 자동 감지하여 `budget_2025_sup` / `budget_2026_req` 정확히 매핑
- **신규사업 판별 방식 전환**: 예산 유무 추측(174건) → PDF "□ 사업 성격" 표의 신규 열 ○ 표시 직접 감지(161건)로 전환, 오판 13건 수정
- **pdfplumber fallback 개선**: 사업명 열 자동 감지 건너뛰기, `"신규"` 토큰 인식 추가
- **5열 구조 `budget_2025_sup`**: 추경 없음을 `None`으로 명확히 처리 (기존 277건 중복값 제거)

## 🚀 실행 방법

### 사전 요구사항

- Python 3.10+
- `uv` 또는 `pip` (패키지 매니저)

### 1. 프로젝트 클론

```bash
git clone https://github.com/<your-username>/AIbudget.git
cd AIbudget
```

### 2. 가상환경 설정 및 의존성 설치

```bash
# uv 사용 시
uv venv --python 3.12
source .venv/bin/activate
uv pip install pandas pdfplumber pymupdf

# pip 사용 시
python -m venv .venv
source .venv/bin/activate
pip install pandas pdfplumber pymupdf
```

### 3. PDF에서 데이터 추출 (선택)
>
> `data/budget_data.csv`가 이미 포함되어 있으므로, PDF 원본이 없으면 이 단계를 건너뛰어도 됩니다.

```bash
python extract_all.py
```

### 4. 대시보드 데이터 생성

```bash
python dashboard/prepare_data.py
```

→ `dashboard/data.json` 파일이 생성됩니다.

### 5. 대시보드 실행

```bash
# Python 내장 서버 사용
cd dashboard
python -m http.server 8000
```

→ 브라우저에서 `http://localhost:8000` 접속

또는 VS Code의 **Live Server** 확장을 사용하여 `dashboard/index.html`을 직접 열 수도 있습니다.

### 6. GitHub Pages로 배포 (선택)

`dashboard/` 폴더를 정적 호스팅 서비스에 바로 배포할 수 있습니다.

```bash
# GitHub Pages 설정 (Settings → Pages → Source → Deploy from a branch)
# Branch: main, Folder: /dashboard
```

또는 **Netlify / Vercel**에서 `dashboard/` 폴더를 루트로 지정하면 즉시 배포됩니다.

## 🛡 주의사항

- PDF에서 자동 추출한 데이터로, **일부 오차가 있을 수 있습니다**.
- `resource/budget_depts/` 폴더의 PDF 파일은 정부 공개 문서이며, 저작권은 해당 기관에 있습니다.
- `file://` 프로토콜로 직접 열면 CORS 제한으로 데이터 로딩이 실패합니다. 반드시 **로컬 서버**를 사용하세요.
