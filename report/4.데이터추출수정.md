## 3. 사업코드 행 혼입 버그 수정

**문제**: `행정서비스통합포털운영(정보화)(2047-500)` 사업에서 PDF가 사업명을 3줄로 분리하여 출력할 때, 마지막 줄인 `(2047-500)` 이 데이터 행으로 인식됨

```
PDF 텍스트 블록:
  "행정서비스통합"   ← _name_key[:6] 매칭으로 건너뜀
  "포털운영(정보화)" ← 수집 (숫자 없음, 무해)
  "(2047-500)"       ← 이게 문제! 2047, 500을 숫자로 추출
```

**결과**: `budget_2024=2047, budget_2025=500` (잘못된 값), 실제 예산 데이터가 2칸 밀림

**수정**: `_collect_data_lines` 내 사업코드 형태 행 필터 추가

```python
# 사업코드 행 건너뛰기: (2047-500), (500), (2047) 형태 모두 처리
if re.match(r"^\(\d[\d\-]*\)$", line):
    continue
```

**범용성**: `^\(\d[\d\-]*\)$` 패턴은 `(2047-500)`, `(500)`, `(2047)` 형태를 모두 커버

## 4. 6열 구조 분기 누락 수정

**문제**: 일부 부처(경찰청 R&D, 소방청 등)가 7열이 아닌 6열 총괄표를 사용함에도, `>= 6` 분기가 없어 `>= 5` 분기로 처리됨

```
실제 6열 데이터: [1760, 1907, 1907, 1705, -202, -10.6]
>= 5 분기 처리:  budget_2026=1907 (틀림), change_amount=1705 (틀림)
>= 6 분기 처리:  budget_2026=1705 (맞음), change_amount=-202 (맞음)
```

**영향**: `change_amount == budget_2026` 인 케이스가 16건 발생 (1칸 밀림 패턴)

**수정**: `extract_budget_summary`와 `extract_budget_by_table` 모두에 `>= 6` 분기 신설

```
7열: 2024결산 | 2025본예산 | 2025추경(A) | 2026요구안 | 2026본예산(B) | 증감(B-A) | 증감률
6열: 2024결산 | 2025본예산 | 추경 또는 요구안     | 2026본예산(B) | 증감(B-A) | 증감률
5열: 2024결산 | 2025본예산 | 2026본예산(B) | 증감(B-A) | 증감률
```

## 5. 신규사업 판별 방식 전환

**기존 방식**: `budget_2024 is None and budget_2025 is None and budget_2026 is not None`

문제 케이스:

| 사업 | 실제 | 구 판별 | 원인 |
|---|---|---|---|
| 국방부 `AI응용제품 신속 상용화 지원사업` | 신규 | 계속 오판 | 전년도 예산 있으나 신규 |
| 금융위원회 `국민성장펀드` | 계속 | 신규 오판 | 전년도 예산 누락 |
| 소방청 `위험물 시설 화재·폭발 예방` | 신규 | 계속 오판 | 동일 |

**새 방식**: PDF "□ 사업 성격" 표의 **신규 열 ○ 표시** 직접 감지

```
┌──────┬──────┬──────┬─────────────────┐
│ 신규 │ 계속 │ 완료 │ 예비타당성 실시 │ ...
├──────┼──────┼──────┼─────────────────┤
│  ○  │      │      │                 │  ← 신규 사업
└──────┴──────┴──────┴─────────────────┘
```

**구현**: `extract_is_new_from_pdf(doc, local_start, local_end)`
- PyMuPDF `get_text("words")`로 단어별 x·y 좌표 획득
- "성격" 단어의 y 위치로 사업 성격 섹션 탐지 (100pt 이내)
- "신규"·"계속" 헤더의 x 중심점 계산
- 헤더 아래의 `"○"` 또는 `"O"`가 "신규" 쪽에 더 가까우면 `True`

**특이사항 처리**:
- `"신규계속완료"` 처럼 헤더가 단일 텍스트로 합쳐진 경우 → 문자 비례로 x 위치 추정
- PDF에 따라 `'○'` (U+25CB) 또는 `'O'` (대문자) 로 추출되는 경우 모두 인식

**결과**: 구 방식(174건) → 신 방식(161건), 오판 13건 수정

## 6. 6열 3번째 열 모호성 해소

**문제**: 6열 구조에는 두 가지 변형이 존재하며, 기존 코드는 항상 3번째 열을 `budget_2025_sup`(추경)으로 매핑

```
6열 A: 2024 | 2025 | 추경(A)  | 2026(B) | 증감 | 증감률  ← 과기부 외 대부분
6열 B: 2024 | 2025 | 요구안   | 2026(B) | 증감 | 증감률  ← 과기부 R&D 일부
```

**영향**: 6열 B 구조에서 `budget_2025_sup`에 2026년 요구안 금액이 들어가는 오염 (12건 이상)

**수정**: 총괄표 헤더 텍스트에서 "추경"/"요구안" 키워드 감지로 구조 자동 판별

```python
header_context = "\n".join(lines[budget_start: min(budget_start + 20, len(lines))])
_has_추경 = "추경" in header_context
_has_요구 = any(kw in header_context for kw in ["요구안", "요구"])

# 6열 분기에서
if _has_요구 and not _has_추경:
    result["budget_2026_req"] = _parse_token(nums[2])  # 요구안
else:
    result["budget_2025_sup"] = _parse_token(nums[2])  # 추경 (기본)
```

## 7. pdfplumber fallback 개선

`extract_budget_by_table` 함수에 두 가지 수정을 적용했습니다.

### 7-1. 사업명 열 처리 불일치

**문제**: 사업명 열이 텍스트인 경우 `pass`(미포함), 빈 셀인 경우 `"DASH"`(포함)로 처리가 달라 `values` 배열 길이가 실제 숫자 열 수와 불일치 발생

**수정**: 헤더 행에서 사업명/목명 열 인덱스를 사전에 감지하고, 데이터 수집 시 일관 건너뛰기

```python
name_col_idx = None
for col_idx, cell in enumerate(table[0]):
    if cell and ("목명" in str(cell) or "사업명" in str(cell)):
        name_col_idx = col_idx
        break

for col_idx, cell in enumerate(data_row):
    if col_idx == name_col_idx:
        continue  # 항상 건너뜀
    ...
```

### 7-2. `"신규"` 토큰 미인식

**문제**: `cell_str in ("순증", "순감")` 처리에 `"신규"` 누락 → 신규 사업 증감률 셀에서 열 매핑 밀림

**수정**: `elif cell_str in ("순증", "순감", "신규"):` 로 확장

## 8. 5열 구조 budget_2025_sup 처리 수정

**기존**: 5열 구조(추경·요구안 없음)에서 `budget_2025_sup = budget_2025` 동일값 복사 (277건)

- 추경이 없는 경우와 추경이 본예산과 같은 경우를 구분 불가

**수정**: `budget_2025_sup = None` 으로 처리

- DB 뷰의 `COALESCE(budget_2025_sup, budget_2025, 0)` 로 기존 집계 로직 유지
- 의미 명확화: `None` = 추경 정보 없음
