## 1. 개편 배경

41개 부처, 533개 AI 관련 예산 사업 데이터를 PDF에서 추출하여 대시보드로 시각화하는 과정에서, 사용자의 피드백에 따라 **데이터 정합성 검증**, **UI 개선**, **기능 확장**을 단계적으로 수행하였다.

## 2. 초기 구현 (v1)

### 기술 스택

- **프론트엔드**: HTML/CSS/JavaScript + Chart.js
- **데이터 파이프라인**: Python(`extract_all.py`) → CSV → JSON(`prepare_data.py`) → Dashboard

### 초기 화면 구성

- KPI 카드 4개 (총 예산, 전체 사업수, 신규 사업, 예산 증감)
- 부처별 예산 규모 바 차트
- 예산 변동 비율 / 신규사업 비율 도넛 차트
- 부처별 상세 분석 (드롭다운 + 연도별 추이 + 사업 목록 테이블)
- Top 10 예산 증가/감소 수평 바 차트

## 3. 데이터 검증 및 오류 수정

### 3.1 CSV → JSON 정합성 검증

- 533건 사업 수, 41개 부처별 사업 수, 부처별 2026 예산 합계 → **100% 일치 확인**

### 3.2 PDF 원본 → CSV 대조 검증

- macOS 파일시스템의 유니코드 NFD/NFC 차이로 초기 PDF 매핑 실패
- `unicodedata.normalize('NFC')` 적용으로 해결
- 10개 부처 / 20건 랜덤 샘플링 → **100% 통과**

### 3.3 발견·수정한 오류

| 항목 | 수정 전 | 수정 후 |
|---|---|---|
| 총 예산 단위 변환 | `/ 10000` (2735.5조원) | `/ 1000000` (27.4조원) |
| 전년 대비 증가액 | `increase_amount` (증가사업 합계: 89,426억) | `net_change` (순증감: 84,974억) |
| 캐시 문제 | 브라우저가 구 data.json 로드 | 캐시 무효화 meta 태그 + 타임스탬프 쿼리 |

### 3.4 금액 단위 변경

- 원본 데이터: **백만원** 단위
- 차트/테이블 표시: 백만원 → **억원** (÷100) 으로 변환
- 모든 차트 Y축, 툴팁, 테이블 헤더에 `단위: 억원` 명시

## 4. 단계별 UI 개선

### 4.1 레이아웃 조정

- 연도별 추이 차트를 부처별 상세 분석 내 `col-span-2`로 가로 전체 폭 배치
- 제목을 "상위 10개 사업"으로 명확화

### 4.2 신규사업 상세 분석 섹션 추가

기존 도넛 차트만으로 부족한 신규사업 정보를 보강:

- 부처별 신규사업 예산 바 차트
- 부처별 신규사업 현황 테이블 (신규 수 / 전체 수 / 비율)
- Top 10 신규사업 수평 바 차트
- 부처별 드롭다운 + 검색 기능 포함 신규사업 목록

### 4.3 부처별 필터 패턴 통일

- "전체 목록" 방식에서 부처별 드롭다운 셀렉트 + 검색 방식으로 변경
- 전체 사업 드릴다운과 동일한 UX 패턴 적용

## 5. 탭 구조 전면 개편 (v2)

### 5.1 개편 동기

신규사업, 예산 증가, 예산 감소 각각에 대한 상세 분석 요구가 증가하면서, 하나의 긴 페이지로는 가독성과 탐색성이 저하. **4개 탭으로 분리**하여 사용자가 관심 영역에 집중할 수 있도록 개편.

### 5.2 탭 구성

```
┌─────────────────────────────────────────────────┐
│  KPI 카드 (상단 고정)                             │
├─────────┬─────────┬─────────┬──────────┤
│ 📊전체  │ 🆕신규  │ 📈증가  │ 📉감소   │  ← 탭 네비게이션
├─────────┴─────────┴─────────┴──────────┤
│  [탭별 콘텐츠 영역]                               │
└─────────────────────────────────────────────────┘
```

| 탭 | 차트 | 테이블 |
|---|---|---|
| **전체사업** | 부처별 예산 바 / 변동비율 도넛 / 신규비율 도넛 | 부처별 드릴다운 (연도별 추이 + 전체 목록) |
| **신규사업** | 부처별 신규 예산 바 / Top10 수평 바 | 부처별 현황 (신규/전체/비율) + 목록 |
| **예산 증가** | 부처별 증가액 바 / Top10 수평 바 | 부처별 현황 (증가/전체/비율) + 목록 (증가액·증가율 포함) |
| **예산 감소** | 부처별 감소액 바 / Top10 수평 바 | 부처별 현황 (감소/전체/비율) + 목록 (감소액·감소율 포함) |

### 5.3 핵심 기술 구현

- **Lazy Rendering**: 탭 클릭 시에만 Chart.js 인스턴스를 생성하여 초기 로딩 성능 최적화
- **공통 헬퍼 함수**: `renderDeptSummaryTable()`, `renderHorizontalBar()`, `setupCategoryTable()`, `renderCategoryTable()` 로 코드 재사용
- **탭별 색상 테마**: 신규=녹색, 증가=파란색, 감소=빨간색
- **반응형 탭**: 768px 이하에서 탭 버튼 2열 배치

### 5.4 수정된 파일

| 파일 | 변경 내용 |
|---|---|
| `dashboard/index.html` | 4개 탭 구조로 전면 재작성, 각 탭별 HTML 섹션 |
| `dashboard/dashboard.js` | 탭 전환 로직 + 4개 탭별 렌더링 함수 + 공통 헬퍼 |
| `dashboard/style.css` | 탭 네비게이션, 카테고리 배지, 반응형 CSS 추가 |
| `dashboard/prepare_data.py` | `net_change`, `total_budget_2025` 필드 추가 |

## 6. UI/UX 고도화 및 정적 호스팅 최적화 (v3)

### 6.1 테마 및 스타일 개선

- **다크/라이트 테마 토글**: CSS 변수(`[data-theme="light"]`)를 도입하여 전역 테마 동적 전환 기능 적용
- `localStorage`에 사용자의 테마 선택 상태를 저장하여 페이지 재방문 시 자동 복구
- 🌙 / ☀️ Bootstrap Icon 버튼 클릭 시 Chart.js 인스턴스의 축, 눈금, 툴팁 색상도 JavaScript 레벨에서 실시간 업데이트
- **커스텀 스크롤바**: 대시보드의 테마 룩 앤 필(Look and Feel)을 해치지 않게 전체 영역(툴팁 내부 포함)에 얇고 모던한 스크롤바 디자인 적용

### 6.2 편의성(UX) 강화

- **상세 사업 내용(Description) 툴팁**: 테이블의 사업명 컬럼 행에 마우스를 올리면(`hover`) 사업 목적 등이 담긴 말풍선 툴팁 컨테이너가 우측에 노출되도록 구현
- 툴팁 유지 브릿지(Bridge): CSS `::before` 가상 요소로 투명한 연결 공간을 확보하여 텍스트 밖으로 마우스가 잠시 나가도 팝업이 사라지지 않고 내용 스크롤·드래그가 가능하도록 조치

### 6.3 배포용 정적 최적화

- 로컬 `file://` 환경에서만 발현되던 CORS `fetch` 및 `data.json?v=` 캐시 타임스탬프 오류 제거
- GitHub Pages, Netlify 등 정적 호스팅 서비스(Static Hosting) 배포에 호환되도록 정리

## 7. 최종 파일 구조

```
dashboard/
├── index.html          # 메인 HTML (탭 구조 + 테마 토글 버튼)
├── dashboard.js        # 차트/테이블 렌더링 로직 + 테마 토글/차트 업데이트 처리
├── style.css           # 다크/라이트 테마 CSS 변수 + 탭/툴팁 스타일
├── prepare_data.py     # CSV → JSON 변환
└── data.json           # 대시보드 데이터 (자동 생성)
```
