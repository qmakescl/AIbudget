## 1. 분석 목적

본 보고서는 `AIbudget` 프로젝트에서 대시보드 구축을 위해 각 부처별 예산 사업 설계서(PDF)로부터 '예산', '사업내용', '주요 성과' 등의 핵심 데이터를 자동으로 추출할 수 있는지 그 타당성과 가능성을 검토하기 위해 작성되었습니다.

## 2. 메타데이터 인덱스 분석

제공된 `00_사업목록_인덱스.json` 파일의 구조를 분석한 결과, 아래와 같은 구조적인 데이터 매핑이 가능함을 확인하였습니다.

- **부처별 PDF 파일 매핑**: 각 부처에 해당하는 분할된 PDF 파일명(`파일명`) 정보 보유
- **페이지 오프셋 정보**: 전체 원본 PDF 기준페이지(`원본_시작페이지`, `원본_끝페이지`)를 기준으로 사업별 페이지가 맵핑되어 있어, 부처별 PDF의 로컬 페이지 번호(Zero-index)로 정확한 환산이 가능합니다.
- **사업 단위 분리**: 사업명 및 사업별 할당 페이지 정보가 매우 구체적으로 작성되어 있어 문서를 분할 추출하는 데 무리가 없습니다.

## 3. PDF 파싱 테스트 결과

Python 기반의 `PyMuPDF(fitz)` 라이브러리를 활용하여 가장 첫 번째 부처인 **"감사원"의 "전산운영경비(정보화)"** 사업을 대상으로 텍스트를 추출하는 파일럿 테스트를 수행하였습니다.

그 결과, 추출된 텍스트에는 다음과 같은 정형화된 패턴이 존재함을 확인했습니다.

- `가. 예산 총괄표`: 연도별 예산 데이터(결산, 예산, 요구안 등)가 표 형태로 존재함
- `나. 사업설명자료`: `1) 사업목적·내용` 등과 같이 사업의 내용과 목적을 나타내는 텍스트 영역이 고정적인 헤더 번호와 함께 명시되어 있음

PDF 특성상 표(Table) 구조나 레이아웃이 완전히 정형화되지 않은 경우도 있으나, 정규표현식이나 `pdfplumber`의 표 추출 기능 혹은 Text 기반의 키워드 마커(e.g., "가. 예산 총괄표", "나. 사업설명자료")를 활용하여 핵심 데이터를 효과적으로 스크래핑할 수 있을 것으로 판단됩니다.

## 4. 결론 및 향후 과제

1. **추출 기능성 확인**: 부처별 사업 설계서에서 사업 내용, 예산 데이터, 주요 성과를 파싱하는 것은 구조적으로 충분히 가능합니다.
2. **텍스트 정제 전략 필요**: 모든 PDF가 완벽히 동일한 형식을 유지하지 않을 수 있다는 점("어떤 사업에는 충실히 기록하지 않은 경우도 있다")을 고려하여, 파싱 결과값이 비어 있거나 헤더가 달라진 경우에 대한 예외 처리 로직(Fallback)을 견고하게 설계해야 합니다.
3. **대시보드 데이터 구성**: 추출된 데이터를 JSON, CSV 또는 데이터베이스 형태로 구조화하여 프론트엔드 대시보드와 연결할 수 있는 파이프라인의 후속 작업이 필요합니다.
