# 대시보드 개선 구현계획서

## 1. 배경 및 목적

현재 대시보드는 4개 탭으로 구성되어 있다:
- **전체사업**: 부처별 예산 규모, 변동 비율, 드릴다운 분석
- **신규사업**: 신규사업 예산/건수 분석
- **예산 증가**: 증가 사업 상세 분석
- **예산 감소**: 감소 사업 상세 분석

이번 개선은 3가지 항목을 포함한다:

### 개선 1: "부처별 현황" 탭 신규 추가
"전체사업" 탭에 부처 드릴다운 기능이 있으나, 특정 부처를 선택해 **예산 구성·사업 분포·연도별 추이를 종합적으로 비교**하는 전용 뷰가 부재함. 부처를 선택하면 해당 부처의 예산 규모, 사업 구성(신규/계속, 증가/감소/동결), 주요 사업 등을 한 화면에서 파악할 수 있는 5번째 탭을 추가한다.

### 개선 2: 사업내용 툴팁 개선 (위치 + 줄바꿈)

현재 사업명 호버 시 나타나는 사업내용 팝업이 셀 오른쪽에 고정 위치(`left: calc(100% + 12px)`)로 표시되어 사업명과 거리가 떨어져 불편함. 이를 **마우스 커서를 따라다니는 방식**으로 변경한다. 또한 사업내용 텍스트에 bullet(○, ●, ◦, - 등) 문자가 연속으로 이어져 가독성이 떨어지는 문제가 있어, **두 번째 bullet부터 앞에 줄바꿈을 삽입**하여 보기 좋게 개선한다.

### 개선 3: "전체사업" 탭에 부처별 산점도 추가
부처별 사업수(X축)와 예산 총액(Y축)의 관계를 시각화하는 **산점도(Scatter Plot)**를 추가하고, 부처 전체 평균을 십자선으로 표시하여 각 부처의 상대적 위치를 직관적으로 파악할 수 있게 한다.

#### (4) 주요 사업 Top 10 (수평 바 차트)

#### (5) 전체 사업 목록 테이블
- 검색 입력창으로 사업명 필터링
- 컬럼: 사업명 | 2024 결산(억원) | 2025 예산(억원) | 2026 예산(억원) | 증감액 | 증감률 | 구분(신규/계속)
- 2026 예산 기준 내림차순 정렬
- 사업명 호버 시 설명 툴팁 (기존 전체사업 탭과 동일)

## 4. 사업내용 툴팁 개선 설계

### 4.1 마우스 추적 방식 변경

**현재 방식** (CSS only):
- `.project-tooltip`이 `position: absolute; left: calc(100% + 12px)` 고정
- `.project-name-cell:hover .project-tooltip`으로 표시/숨김

**개선 방식** (JS `mousemove` 이벤트):
- CSS의 hover 기반 표시를 제거하고, JS로 `mouseenter`/`mouseleave`/`mousemove` 이벤트 처리
- 툴팁을 `position: fixed`로 변경하여 마우스 커서 근처(우측 하단 방향, 약 12px offset)에 표시
- 뷰포트 경계 감지: 오른쪽/아래쪽 경계에 닿으면 반대 방향으로 자동 전환
- 기존 `::before` 브릿지와 `::after` 화살표는 제거 (커서 추적이므로 불필요)

### 4.2 사업내용 줄바꿈 처리

**현재 상태**: 사업내용이 `○항목1 ○항목2 ○항목3`처럼 한 줄로 이어짐

**개선**: 두 번째 bullet부터 앞에 `\n` 삽입
- 대상 문자: `○`, `●`, `◦`, `-`, `·`, `▪`, `◆`, `▶` 등
- 처리 위치: 테이블 렌더링 시 description 텍스트 전처리
- 정규식: 첫 bullet 이후 등장하는 bullet 앞에 줄바꿈 삽입
  ```js
  // 예시: 두 번째 이후 bullet 앞에 줄바꿈
  desc.replace(/(?<=.)\s*([○●◦\-·▪◆▶])/g, '\n$1')
  ```
- 툴팁의 기존 `white-space: pre-wrap` 속성이 줄바꿈을 그대로 반영

### 4.3 수정 파일

| 파일 | 변경 내용 |
|------|----------|
| `dashboard/style.css` | `.project-tooltip` position 변경 (absolute → fixed), hover 표시 규칙 제거, `::before`/`::after` 제거 |
| `dashboard/dashboard.js` | `mousemove` 이벤트 핸들러 추가, description 줄바꿈 전처리 함수 추가 |

## 5. "전체사업" 탭 산점도 설계

### 5.1 차트 사양

- **차트 유형**: Chart.js `scatter` 타입
- **위치**: "전체사업" 탭의 기존 차트 영역(부처별 예산 규모 바 차트와 도넛 차트 사이 또는 아래)에 full-width로 추가
- **X축**: 사업 수 (건)
- **Y축**: 2026 예산 총액 (억원)
- **데이터 포인트**: `dept_summary` 41개 부처 각각 1개 점

### 5.2 평균선 표시

- **X축 평균선**: 부처별 사업수 평균 (세로 점선)
- **Y축 평균선**: 부처별 예산액 평균 (가로 점선)
- Chart.js `annotation` 플러그인 사용 또는 커스텀 플러그인으로 구현
- 선 색상: `colors.gray` + 점선 스타일, 라벨로 "평균 N건" / "평균 N,NNN억원" 표시

### 5.3 툴팁 (마우스 오버 시)

각 점(부처) 위에 마우스 오버 시 다음 정보 표시:

| 항목 | 데이터 소스 | 표시 형식 |
|------|-----------|----------|
| 부처명 | `dept_name` | 타이틀 |
| 사업 수 | `project_count` | `N건` |
| 예산 총액 | `budget_2026_mil` | `N,NNN 억원` |
| 신규사업 수 및 비율 | `new_count` / `project_count` | `N건 (N.N%)` |
| 증가 예산액 | `dept_summary`에서 증가 사업 합산 | `+N,NNN 억원` |
| 감소 예산액 | `dept_summary`에서 감소 사업 합산 | `-N,NNN 억원` |

> 증가/감소 예산액은 `data.projects`에서 해당 부처 사업을 필터링하여 합산하거나, `prepare_data.py`에서 `dept_summary`에 미리 포함할 수 있음. 현재 `dept_summary`에 `increased_count`/`decreased_count`는 있으나 증감 **금액** 합계는 없으므로, JS에서 `projects` 배열 기반으로 실시간 집계.

## 6. 사후 추가 개선: 산점도 고도화 및 사업수별 부처 현황 표

계획서 작성 이후 산점도와 관련하여 아래 추가 개선을 구현하였다.

### 6.1 산점도 제목·설명 개편

| 항목 | 이전 | 이후 |
| --- | --- | --- |
| 제목 | 부처별 사업수 vs 예산 규모 | 사업수와 예산규모 |
| 부제 | 각 점 = 1개 부처, 점선 = 전체 평균 \| 양 축: 로그 스케일... | 피어슨 상관계수 r = 0.XXX \| 양 축: 로그 스케일... |

- 부제의 `<span id="scatterCorrLabel">` 요소를 JS가 초기화 시점에 채움

### 6.2 평균선 색상 변경 및 중앙값선 추가

- **평균선**: 기존 회색(`rgba(100,116,139,…)`) → 붉은 계열 (`rgba(239,68,68,…)`)
- **중앙값선 신규 추가**: 노란 계열 (`rgba(234,179,8,…)`)
  - `median()` 함수로 사업수·예산액의 중앙값 각각 계산
  - 세로선: 사업수 중앙값, 가로선: 예산액 중앙값
  - 라벨: 평균 라벨(top+14px) 아래에 중앙값 라벨(top+28px) 배치하여 겹침 방지

### 6.3 피어슨 상관계수 추가

- 산점도 데이터(`scatterPoints`)에서 피어슨 r 계산

  ```text
  r = Σ((xi−x̄)(yi−ȳ)) / √(Σ(xi−x̄)² · Σ(yi−ȳ)²)
  ```

- 계산값을 제목 부제(`#scatterCorrLabel`)에 동적 삽입

### 6.4 산점도 우하단 범례 추가

`avgLinesPlugin.afterDraw()`에서 canvas 우하단에 범례 박스 직접 렌더링:

- 반투명 배경 (`rgba(15,23,42,0.55)`, `roundRect` 6px)
- 행 1: 붉은 점선 ─ `평균 (사업수 N건 / 예산 N억)`
- 행 2: 노란 점선 ─ `중앙값 (사업수 N건 / 예산 N억)`
- 위치: `xs.right − boxW − 8`, `ys.bottom − boxH − 8` (축 영역 기준)

### 6.5 사업수별 부처 현황 표 추가

산점도 아래에 전체 너비 테이블 신규 추가 (`col-span-2`).

#### 컬럼 구성

| 컬럼 | 설명 |
| --- | --- |
| 사업수 | N건 |
| 부처 수 | 해당 사업수를 가진 부처 수 |
| 부처명 | 쉼표 구분, 각 부처명에 커스텀 툴팁 |
| 예산 평균 (천억원) | 해당 그룹 부처의 2026 예산 평균 |

#### `buildScatterDistTable(data)` 구현

- `dept_summary`를 `project_count` 기준으로 그룹화, 오름차순 정렬
- 부처명별 `<span class="dept-tip">` 생성, JS 이벤트 리스너 부착

#### 부처명 커스텀 툴팁 (`#dept-tooltip`)

- `position: fixed` div를 `<body>` 끝에 삽입
- `mouseenter`: 해당 부처의 상세 정보(사업수, 예산액, 사업별 평균예산액, 신규사업수·비율) 렌더링 후 표시
- `mousemove`: 커서 위치 + 12px offset으로 위치 갱신, 뷰포트 경계 보정
- `mouseleave`: 숨김

## 7. 구현 단계

### Step 1: 사업내용 툴팁 개선 (`style.css` + `dashboard.js`)

1. **CSS 변경** (`style.css`):
   - `.project-tooltip`: `position: absolute` → `position: fixed`, `left`/`top`/`transform` 제거
   - `.project-name-cell:hover .project-tooltip` 규칙 제거 (JS로 제어)
   - `.project-tooltip::before` (브릿지) 제거

## 7. 데이터 의존성

기존 `data.json`의 데이터 구조로 완전히 구현 가능:

- **`dept_summary[]`**: 부처별 집계 데이터 (예산, 사업수, 신규/증가/감소/동결 건수)
- **`projects[]`**: 개별 사업 데이터 (연도별 예산, 증감, 사업내용 등)

`prepare_data.py` 수정이나 추가 데이터 가공은 불필요함.
산점도 툴팁의 증가/감소 예산액은 `projects` 배열에서 JS로 실시간 집계.
