## 1. 추출 파이프라인 개요

`extract_all.py`는 `00_사업목록_인덱스.json`을 기반으로 41개 부처 533개 사업 PDF에서 데이터를 추출합니다.

```
00_사업목록_인덱스.json
  → 부처별 PDF 파일 탐색
    → 각 사업 페이지 범위에서 텍스트 추출 (PyMuPDF)
      → 예산 총괄표 파싱 (extract_budget_summary)
        → fallback: pdfplumber 표 추출 (extract_budget_by_table)
      → 사업목적·내용 추출 (extract_description)
      → 신규 사업 판별 (extract_is_new_from_pdf)
  → CSV 및 SQLite DB 저장
```

## 2. 사업명 처리

### 2.1 JSON 인덱스와 PDF의 사업명

| 위치 | 형식 | 예시 |
|---|---|---|
| `00_목차.pdf` (목차 페이지) | 길면 말줄임 | `AI 기반 민군 유무인 복합체계 주파수 공동사용 및 운용…` |
| `00_사업목록_인덱스.json` | 완전 복원된 사업명 | `AI 기반 민군 유·무인복합체계 주파수 공동사용 및 운용 기술 개발` |
| 부처별 PDF 총괄표 첫 번째 열 | 사업명 + (코드-코드) | `정보체계개발유지` 다음 줄에 `(2131-302)` |

**확인 결과**: JSON 인덱스는 이미 완전한 사업명을 보유하고 있으며, 말줄임이 존재하지 않습니다. ✓

### 2.2 사업명 포함 숫자의 열 데이터 밀림 방지

예산 총괄표 첫 번째 열에 사업명이 분리 출력될 때 사업명 내 숫자(`3.0`, `112` 등)가
예산 금액으로 잘못 인식되는 문제를 두 가지 방법으로 방어합니다.

**방어 1: 정수·소수 모두 `_name_numbers`에 등록**

```python
_name_numbers = set(re.findall(r"\d+", project_name))
for _t in re.findall(r"[\d,]+\.?\d*", project_name):
    _cleaned = _t.replace(",", "")
    if _cleaned:  # 빈 문자열 제외
        _name_numbers.add(_cleaned)
```

- `닥터앤서3.0` → `_name_numbers = {"3", "0", "3.0"}` → 토큰 `"3.0"` 필터링 ✓

**방어 2: 역방향 부분 문자열 체크 (숫자 포함 줄)**

```python
line_key = re.sub(r"[\s\(\)（）]", "", line)
if len(line_key) >= 4 and line_key in _name_key and re.search(r"\d", line_key):
    continue
```

- `"조성(닥터앤서3.0)"` → `"조성닥터앤서3.0"` ⊂ `_name_key` AND 숫자 있음 → 스킵 ✓
- `"거래, 평가) 지원"` → 숫자 없음 → 스킵하지 않음 (지식재산처 처리 유지)

**방어 3: `(코드-코드)` 형태 사업코드 행 스킵**

```python
if re.match(r"^\(\d[\d\-]*\)$", line):
    continue
```

- `(2131-302)`, `(508)`, `(2047-500)` 등 → 스킵 ✓

## 3. 신규 사업 구분

### 3.1 PDF 위치 기반 감지

사업 성격 표의 "신규" 헤더 열 x좌표와 ○ 마커의 x좌표를 비교합니다.

```python
# '신규계속완료'처럼 합쳐진 단어도 문자 비례로 열 위치 추정
sg_x_mid, sg_y_bottom = _col_x_mid(table_words, "신규")
gs_x_mid, _ = _col_x_mid(table_words, "계속")

# ○, O(대문자), 0(숫자영) 모두 감지
circle_candidates = [
    w for w in table_words
    if w[4] in ("○", "O", "0") and w[1] >= sg_y_bottom - 2
]

for c in circle_candidates:
    c_x_mid = (c[0] + c[2]) / 2
    dist_to_sg = abs(c_x_mid - sg_x_mid)
    if dist_to_sg < dist_to_gs:  # 신규 열에 더 가까우면
        return True
```

- `"0"` 추가: 일부 PDF에서 PyMuPDF가 ○를 숫자 0으로 추출하는 경우 대응

### 3.2 수동 확인 필요 4건

is_new=True로 감지되었으나 전년도 예산(budget_2025)이 존재하는 사업입니다.
PDF 좌표 기반으로는 "신규" 열에 ○가 표시된 것으로 확인되나, 예산 이력을 고려하면
사업 재편(구조개편 등)에 따른 경우일 수 있습니다.

| 부처 | 사업명 | 2024 | 2025 | 2026 | 비고 |
|---|---|---|---|---|---|
| 국토교통부 | 자율주행글로벌혁신클러스터(R&D) | - | 3,400 | 3,400 | 2025부터 시작 |
| 국토교통부 | 지역특화형자율주행서비스(R&D) | - | 4,500 | 4,500 | 2025부터 시작 |
| 산업통상부 | 국가로봇테스트필드사업(R&D) | 5,600 | 38,131 | 57,655 | 급증, 재편 가능 |
| 행정안전부 | 재난및안전관리연구개발(R&D) | 32,940 | 31,744 | 26,766 | 감소, 계속사업 의심 |

## 4. 예산 데이터 표 파싱

### 4.1 표 구조

| 열 | 내용 | 변수명 |
|---|---|---|
| 1 | 사업명(목명) — 파싱 시 무시 | — |
| 2 | 2024년 결산 | `budget_2024` |
| 3 | 2025년 본예산(A) | `budget_2025` |
| 4 | 2025년 추경(A) **또는** 2026년 요구안 | `budget_2025_sup` / `budget_2026_req` |
| 5 | 2026년 요구안 **(7열)** / 2026년 조정(B) **(6열)** | `budget_2026_req` / `budget_2026` |
| 6 | 2026년 조정(B) **(7열)** / 증감(B-A) **(6열)** | `budget_2026` / `change_amount` |
| 7 | 증감(B-A) **(7열)** / 증감률 **(6열)** | `change_amount` / `change_rate` |
| 8 | 증감률 **(7열만)** | `change_rate` |

- 열 수는 헤더 텍스트에서 "추경"/"요구" 유무로 자동 판별
- 증감 표기: `순증`, `순감`, `신규` → None 처리; `-` → None (예산 없음)
- `△` → 음수

### 4.2 텍스트 추출 전략

```
1차: PyMuPDF 텍스트 파싱 (사업설명자료 break, 60줄 범위)
2차: 1차 실패 시, break 없이 한글 15자 필터 (80줄 범위)
3차: 2차도 실패 시, pdfplumber 표 추출 fallback
```

### 4.3 주요 필터 규칙

| 규칙 | 대상 | 효과 |
|---|---|---|
| `_is_header_line` | 연도, 결산, 예산, 월말 등 키워드 행 | 헤더 메타 행 필터링 |
| `□` break (수집 후) | `□기능별`, `□내역사업별` 등 | 하위 분류표 혼입 방지 |
| **`○` break (수집 후)** | `○기능별 분류(계)` 등 | 집행현황 표 숫자 혼입 방지 |
| `_is_year_number` | 2020-2029 범위 4자리 수 | 연도 숫자 필터링 |
| `_name_numbers` | 사업명 내 숫자 토큰 | 사업명 숫자 오인식 방지 |
| `^\(\d[\d-]*\)$` | `(2131-302)`, `(508)` 등 | 사업코드 행 필터링 |

### v8: 사업명 소수 토큰 처리 (2026-03-08)

**문제**: `닥터앤서3.0` 사업명에서 `"3.0"` 토큰이 예산금액으로 오인
**원인**: `_name_numbers = {"3", "0"}` 이었으나 추출 토큰은 `"3.0"` (소수)
**수정**: `_name_numbers`에 소수 형태(`"3.0"`)도 추가; 빈 문자열 가드 추가

### v9: 역방향 부분 문자열 체크 (2026-03-08)

**문제**: 긴 사업명이 2줄로 분리될 때 두 번째 줄이 스킵되지 않음
**수정**: `line_key in _name_key AND 숫자 있음` 조건으로 역방향 체크 추가

### v10: 집행현황 표 혼입 방지 (2026-03-08)

**문제**: 국방부 `정보체계개발유지(정보화)` 에서 `budget_2026=-4718` 오류 발생
**원인**:
1. 총괄표 직후 집행현황 표(`2025('25.7월말)` 등 날짜 헤더 포함)가 PyMuPDF 순서로 연결 출력
2. `2025('25.7월말)` 에서 `"25.7"` 토큰이 예산금액으로 추출됨 → 7개 숫자 → 7열 매핑 오배분
3. `○기능별 분류(계)` 이후 쪼개진 숫자(`188,3`, `11` 등)도 추가 혼입

**수정 3가지**:

1. `"월말"` → `_HEADER_KEYWORDS` 추가 → `2025('25.7월말)` 행 필터링
2. `○` break 조건을 `_is_header_line` **앞**에 배치 → `○기능별 분류(계)` 행이 "기능별" 키워드로 미리 필터되던 버그 수정
3. `is_new` 감지에 `"0"` (숫자 영) 추가

**결과**: `budget_2026=183,056 백만원` 정상 복원 (이전: -4,718)

## 7. 잔여 이슈

### 7.1 범정부 인공지능 공통기반 구현(정보화) — PDF 구조 이슈

행정안전부 해당 사업의 증감(B-A) 열에 PDF 원본이 `budget_2026`과 동일한 값을 표기.
이는 코드 버그가 아닌 PDF 원본의 비표준 구조이므로 자동 보정 불가.

| 항목 | 현재값 | 정확한 값 |
|---|---|---|
| budget_2026 | 7,371 | 7,371 ✓ |
| change_amount | 7,371 | 1,979 |
| change_rate | 1,979 | ~36.7% |

### 7.2 is_new 수동 확인 필요 4건

[3.2절](#32-수동-확인-필요-4건) 참조.

### 7.3 change_amount NULL 동결 15건

PDF 원본이 증감 열에 `-`를 사용하는 경우 `change_amount=None` 으로 저장됩니다.
`budget_2025 == budget_2026`인 경우 change=0으로 후처리 가능하나,
PDF 원본 값을 충실히 반영하는 현재 방식도 유효합니다.
