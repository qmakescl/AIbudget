# 2026 정부 AI 예산 PDF 추출 — 방식 비교 및 구현 과정

## 개요

41개 부처 PDF에서 533개 AI 관련 사업의 예산 총괄표 데이터를 추출하는 파이프라인을 2회 구현했다.

| 구분 | 파일 | 비고 |
|------|------|------|
| v1 (구) | `extract_all.py` | 원본 추출 코드 |
| v2 (현) | `verify_extract.py` | 재추출 + 자동 검증 |

---

## 1. 두 버전의 핵심 차이

### 1.1 추출 우선순위

```
v1 (extract_all.py)          v2 (verify_extract.py)
────────────────────          ─────────────────────────
1차: PyMuPDF 텍스트 파싱      1차: pdfplumber 표 추출
2차: pdfplumber 표 fallback   2차: PyMuPDF 텍스트 pass1 (60줄, break)
                              3차: PyMuPDF 텍스트 pass2 (80줄, no-break)
                              교차 검증: 두 방법 결과 비교
```

**v1의 문제**: PyMuPDF는 텍스트를 단순 행 순서로 읽어 같은 페이지에 두 개의 표가 있으면 행이 섞인다.
**v2의 해결**: pdfplumber는 셀 경계를 보존하므로 표가 섞이지 않는다. PyMuPDF는 fallback으로만 사용.

### 1.2 열 구조 감지 방식

#### v1 — 위치 고정 (숫자 개수로 구조 추정)
```python
if len(raw_numbers) >= 7:      # 무조건 7열로 가정
    budget_2024    = nums[0]   # 항상 첫 번째 = 2024결산
    budget_2025_sup= nums[2]   # 항상 세 번째 = 추경
    budget_2026    = nums[4]   # 항상 다섯 번째 = 2026예산
```
→ 열 순서가 달라지면 열 밀림 발생 위험.

#### v2 — 의미 기반 (헤더 텍스트 분석)
```python
def detect_column_structure(header_rows):
    if re.search(r"\(B-A\)/A", text):          → change_rate
    elif re.search(r"B-A|증감", text):          → change_amount
    elif re.search(r"2024|결산", text):         → budget_2024
    elif re.search(r"추경", text):              → budget_2025_sup
    elif re.search(r"요구|요청", text):         → budget_2026_req
    elif re.search(r"(조정|본예산).*(B)", text): → budget_2026
```
→ 헤더 텍스트에서 각 열의 의미를 판별하므로 부처별 변형 구조에 강함.

### 1.3 신규사업 판별 방식

| 구분 | 방법 | 특징 |
|------|------|------|
| v1 | PDF 좌표 분석 — "신규" 열 헤더의 x 위치와 ○ 표시의 x 거리 비교 | 레이아웃 의존, 정밀하나 fragile |
| v2 | 예산값 기반 — 2024·2025 예산이 모두 None이면 신규 | 레이아웃 독립적, 안정적 |

v1의 좌표 분석은 PDF마다 렌더링이 달라지면 오판한다. v2는 "과거 예산이 없으면 신규"라는 도메인 규칙을 직접 사용한다.

### 1.4 `○` 행(소계 행) 처리

```
v1: ○로 시작하는 행을 만나면 즉시 break → 단일 총괄행이 ○로 시작하면 데이터 소실

v2: ○ 행을 bullet_rows로 따로 보관
    일반 data_rows가 비어있으면 bullet_rows를 fallback으로 사용
    → 과기부 기금사업("○ 디지털분야글로벌인재양성") 정상 추출
```

이 차이로 v1에서 실패했을 1건(과학기술정보통신부 "디지털분야글로벌인재양성(R&D)")이
v2에서 정상 추출되어 **533/533 (100%)** 완전 추출 달성.

### 1.5 음수 기호 처리

| 기호 | Unicode | 사용 부처 | v1 | v2 |
|------|---------|-----------|----|----|
| △ | U+25B3 | 대부분 | ✓ | ✓ |
| ▽ | U+25BD | 대부분 | ✓ | ✓ |
| ▵ | U+25B5 소삼각형 | 경찰청, 방위사업청 | ✓ | ✓ (v2 업데이트 후) |
| ∆ | U+2206 증분기호 | 산업통상자원부 | ✓ | ✓ (v2 업데이트 후) |

v2 초기 버전은 `△▽` 2종만 처리해 경찰청·방위사업청·산업통상부 일부 음수값이 양수로 잘못 저장될 수 있었다. v1 코드를 참조해 4종으로 확장했다.

## 2. v2 핵심 로직 상세

### 2.1 3단계 Fallback 체인

```
[1단계] pdfplumber 표 추출
  is_budget_summary_table() 조건:
    - "사업명" 또는 "목명" 포함
    - "2026" 포함
    - "예산|결산|증감" 포함
  성공률: 527/533 (98.9%)

  ↓ 실패 시 (표 구조가 없거나 인식 불가)

[2단계] PyMuPDF 텍스트 pass1
  - "가. (예산|지출계획|예산안|지출계획안) 총괄표" 패턴으로 시작점 탐지
  - "나. 사업설명자료" 또는 "□기능별" 에서 종료
  - 60줄 범위
  성공률: 식품의약품안전처 6건 등 pdfplumber 실패 분 처리

  ↓ 숫자 6개 미만 시

[3단계] PyMuPDF 텍스트 pass2
  - 종료점 제한 없음, 80줄
  - 한글 15자 이상 행 필터로 설명 문장 제거
```

### 2.2 의미 기반 열 매핑 우선순위

열 매핑 시 `setdefault` (먼저 감지된 열 유지) 를 사용하며, 처리 순서가 중요:

1. `change_rate` 먼저 — `(B-A)/A` 포함 시 (괄호 이스케이프 `\(B-A\)/A` 필수)
2. `change_amount` — `B-A|증감` 포함 (이미 비율로 분류된 열 제외)
3. `budget_2024` — `2024|결산`
4. `budget_2025_sup` — `추경`
5. `budget_2025` — `본예산|계획` (2026 아닌 것)
6. `budget_2026_req` — `요구|요청`
7. `budget_2026` — `(조정|본예산|확정)+(B)` 또는 마지막 `2026` 열

> **버그 이력**: `(B-A)/A` 를 이스케이프 없이 쓰면 regex가 `B-A/A` (괄호 없음)를 매칭. 결과적으로 change_rate 열이 change_amount로 잘못 분류되어 전체 change_rate=NaN 버그 발생. `\(B-A\)/A` 로 수정.

### 2.3 헤더 행 vs 데이터 행 분리

```python
numeric_ratio = numeric_cells / nonempty_cells
if numeric_ratio >= 0.5 and found_header_end:
    → 데이터 행 (예산 숫자 행)
else:
    → 헤더 행 (열 이름 행)
    found_header_end = True  if "2026|증감|정부안" in row
```

단, ○로 시작하는 행은 `bullet_rows`로 별도 보관 후 일반 데이터 행이 없으면 사용.

### 2.4 2025 실제 예산 결정

```
7열 구조 (추경 열 별도 존재):
  budget_2025_sup = 추경(A) 값  ← 실질 2025 예산 (증감 계산 기준)
  budget_2025     = 본예산 값

6열 구조 (추경 열 없음):
  budget_2025_sup = budget_2025  ← 본예산(A)가 증감 계산 기준
```

### 2.5 신규사업 change_amount 보정

```python
# 신규사업은 전년도 예산이 없어 증감액 = 2026 예산 전액
if result["is_new"] and result.get("change_amount") is None:
    result["change_amount"] = result.get("budget_2026")
```

v1에서는 신규사업의 change_amount가 None으로 남아 대시보드 집계 오류 발생. v2에서 수정.

## 3. 자동 검증 체계

| ID | 검증 항목 | 로직 | 심각도 |
|----|----------|------|--------|
| V1 | 2026 예산값 존재 | `budget_2026 is not None` | ERROR |
| V2 | 산술 일치 | `budget_2026 - actual_2025 ≈ change_amount` (±1) | WARN |
| V3 | 비율 일치 | `(change/actual_2025)×100 ≈ change_rate` (±0.5%) | WARN |
| V4 | 신규 일관성 | 신규인데 2024/2025 예산이 있으면 경고 | WARN |
| V5 | 비율 범위 | `change_rate` ∈ [-100%, 50000%] | WARN |
| V6 | 음수 예산 | 모든 예산값 ≥ 0 | ERROR |

V2·V3 불일치(38+31건)는 추출 오류가 아닌 **부처별 기준 차이**:
- 일부 부처는 `본예산(A)`를 기준으로 증감 계산
- 일부 부처는 `추경(A)`를 기준으로 증감 계산
→ 같은 사업이라도 어느 값을 `actual_2025`로 쓰느냐에 따라 산술이 달라짐.

## 4. 부처별 엣지케이스 이력

| 부처 | 형식 | 처리 방법 |
|------|------|----------|
| 경찰청 | 6열 구조 (추경 없음) | detect_column_structure로 자동 감지 |
| 과학기술정보통신부 (기금) | 총괄행이 `○`로 시작 | bullet_rows fallback |
| 식품의약품안전처 | pdfplumber 표 인식 실패 | PyMuPDF pass1으로 처리 |
| 방위사업청 | `□ 예산 총괄표` (가. 없음) | `_TABLE_TITLE_RE` 패턴에 `□\s*` 포함 |
| 기상청 | `가. 예산안 총괄표` | 패턴에 `예산안` 포함 |
| 과학기술정보통신부 (R&D) | `가. 지출계획 총괄표` | 패턴에 `지출계획` 포함 |
| 경찰청·방위사업청 | `▵`(U+25B5) 음수 기호 | parse_number에 ▵ 추가 |
| 산업통상자원부 | `∆`(U+2206) 증분기호 | parse_number에 ∆ 추가 |
