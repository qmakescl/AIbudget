## 3. v1 → v2: 총괄표 헤더 변형 인식

**문제**: "가. 예산 총괄표" 만 인식하여 74건이 누락됨
**원인**: 부처별로 총괄표 제목이 다름

| 사용된 총괄표 제목 | 부처 예시 |
| --- | --- |
| 가. 예산 총괄표 | 대부분의 부처 |
| 가. 지출계획 총괄표 | 과기정통부 R&D, 기후에너지환경부 |
| 가. 예산안 총괄표 | 기상청 등 |

**수정**: 정규식 확장

```python
# Before
re.search(r"예산\s*총괄표", line)
# After
re.search(r"(예산|지출계획|예산안|지출계획안)\s*총괄표", line)
```

## 4. v2 → v3: '목명' 헤더 및 확장 키워드

**문제**: 일부 사업의 총괄표에서 열 이름이 "사업명" 대신 "목명" 사용
**원인**: `_HEADER_KEYWORDS`에 '목명', '계획', '정부안', '확정' 등이 누락
**수정**: `_HEADER_KEYWORDS`에 키워드 추가 + 꺾쇠/주석 메타 행(`<이관 내역>`, `※`) 필터링

```python
_HEADER_KEYWORDS = [

    "사업구조", "구조개편", "기능별", "내역사업",
    "현액", "집행액", "불용액", "이월액", "당초", "수정",

```

## 5. v3 → v4: 2-Pass 방식 도입

**문제**: PyMuPDF가 텍스트 블록을 위치 기반으로 추출할 때, 표 데이터가 "나. 사업설명자료" 아래에 출력되는 경우 존재
**원인**: PDF 레이아웃에서 표가 특정 위치에 있으면 텍스트 추출 순서가 시각적 순서와 다름

```
시각적: 총괄표 헤더 → 데이터 행 → 나. 사업설명자료
PyMuPDF: 총괄표 헤더 → 나. 사업설명자료 → 데이터 행 (뒤집힘!)
```

**수정**: 2-Pass 전략

- **1차**: `"사업설명자료"` break 유지 (정확도 우선, 60줄 범위)
- **2차** (1차 실패 시): break 없이 한글 15자 이상 문장 필터로 확장 탐색 (80줄)

```python
data_lines = _collect_data_lines(budget_start + 1, 60, use_break=True)
raw_numbers = _extract_numbers(data_lines)
if len(raw_numbers) < 2:
    data_lines = _collect_data_lines(budget_start + 1, 80, use_break=False)
    raw_numbers = _extract_numbers(data_lines)
```

## 6. v4 → v5: 하이픈 '-' 인식 및 □ break

**문제**: 신규사업은 2024/2025 열에 `-` (예산 없음)로 표기되지만, 숫자 정규식이 `-` 를 인식하지 못함
**결과**: `[20000, 20000, 순증, 순증]` = 4개만 추출 → 7개 미만 → 실패

**수정**:

- `-` 단독 행을 `DASH` 마커로 인식 → `None`으로 변환
- `□기능별` 이후의 하위 내역 데이터 혼입 방지를 위해 `□` break 추가

```python
if line == "-":
    nums.append("DASH")
    continue
```

## 7. v5 → v6: pdfplumber fallback 도입

**문제**: PyMuPDF 텍스트 추출 순서 문제가 근본적으로 해결 불가능한 경우 존재
**대표 사례**: 고용노동부 "미래환경변화 대응 산업안전보건 연구개발(R&D)"

```
시각적:  총괄표 → 데이터 행 → □기능별
PyMuPDF: 총괄표 → □기능별 헤더 → 하위표 데이터 → 총괄표 데이터 (완전 뒤섞임)
```

**수정**: `extract_budget_by_table()` 함수 추가

- PyMuPDF 1차/2차 실패 시 **pdfplumber**의 `extract_tables()`로 시각적 표를 직접 읽음
- `"목명"` 또는 `"사업명"` 헤더가 있고 `"2026"` 연도가 포함된 표를 찾아 데이터 추출

```python
if budget["budget_2026"] is None:
    budget = extract_budget_by_table(pdf_path, local_start, local_end)
```

추가 개선:

- `"목명"/"사업명"` 서브 앵커 검색으로 데이터 수집 시작점 정확히 찾기
- `○`, `․` 접두어 행 필터링 (기능별 분류 합계 등)

## 8. v6 → v7: '신규' 키워드 인식

**문제**: 일부 신규사업에서 증감률 열에 `"순증"` 대신 `"신규"` 표기
**대표 사례**: 과학기술정보통신부 "AI·SW 중심대학"
**결과**: `[DASH, DASH, DASH, 117960, 117960, 117960]` = 6개 → 7개 미만 → 실패

**수정**: `num_pattern` 정규식에 `"신규"` 추가

```python
# Before
num_pattern = re.compile(r"[△▽]?[\d,]+\.?\d*|순증|순감")
# After
num_pattern = re.compile(r"[△▽]?[\d,]+\.?\d*|순증|순감|신규")
```

## 9. 핵심 교훈

- **PDF 텍스트 추출 ≠ 시각적 순서**: PyMuPDF의 `get_text()`는 텍스트 블록을 위치 기반으로 추출하므로 표 데이터의 순서가 시각적 순서와 다를 수 있음
- **Fallback 전략**: 단일 방법으로 100%를 달성하기 어려우므로 다단 fallback 구조(텍스트 파싱 → 확장 탐색 → 표 추출)가 효과적
- **데이터 다양성**: 예산 없음(`-`), 증감률 표기(`순증`, `순감`, `신규`), 헤더 명칭(`사업명`, `목명`) 등 PDF마다 표기가 다름
- **점진적 개선**: 실패 사례를 하나씩 분석하고, 기존 정확도를 유지하면서(회귀 테스트) 점진적으로 커버리지를 확대하는 것이 중요
